DESIGN PROJECT  
 
 
1. Description  
In this project, you are required to create a caching proxy server that forwards requests from a client 
to a target server and relays back the response. It could include features like caching, blocking certain 
URLs, and logging requests.  
 
2. Requirements  
Write a python code to implement the following functions in your proxy server:  
 
A. Basic Proxy Functionality : 
o Accept requests from clients and forward these requests to a specified target server.  
o Retrieve responses from the target server and relay them back to the client.  
o Support both HTTP and as a BONUS  HTTPS requests, with HTTPS being forwarded 
securely without decrypting.  
B. Socket Programming : 
o Use sockets to manage connections between the client and the target server.  
o Set up the server to listen for incoming client requests on a designated port.  
o Establish a connection to the target server and facilitate data transfer between the 
two.  
C. Request Parsing : 
o Parse client requests to extract information such as the target server's address, port, 
and request type (GET, POST, etc.).  
o Modify headers if necessary to comply with proxy forwarding requirements (e.g., 
setting the "Host" header or stripping other client -specific headers).  
D. Threading : Handle multiple client requests concurrently using multithreading. This allows 
the proxy server to manage multiple connections simultaneously.  
E. Logging : Implement logging to record details of requests and responses, such as:  Client IP 
address and port ; Target server address and port ; HTTP method and URL requested ; 
Timestamp of request and response ; and Log error messages if requests fail.  
F. Content Caching : 
o Cache responses from the target server so repeated requests for the same resource 
can be served from the cache, reducing server load.  
o Implement cache invalidation rules based on response headers (e.g., Cache -Control, 
Expires) or a custom cache timeout.  

G. Blacklist/Whitelist : 
o Add URL or IP filtering by maintaining a list of allowed (whitelisted) or disallowed 
(blacklisted) domains or IP addresses.  
o Reject requests to blacklisted resources with a custom response.  
H. HTTPS Proxy ( BONUS ): 
o If you want to intercept and inspect HTTPS traffic (rather than just forward it), 
implement SSL/TLS decryption using an SSL certificate.  
o This requires setting up a man -in-the-middle (MITM) structure with a self -signed 
certificate, which allows the proxy to decrypt HTTPS traffic.  
o Be cautious with this as it impacts user privacy and security —this feature is 
generally used for debugging or educational purposes.  
I. Admin Interface  (BONUS) : 
o Build a web -based admin interface for viewing and managing logs, cache entries, 
and blacklist/whitelist configurations.  
o Provide basic stats on proxy usage, cache performance, and active connections.
